# TraceMind — Deployment Guide

Stack: **Supabase** (Postgres) → **Render** (FastAPI) → **Vercel** (Next.js)

All code is deployment-ready. No source changes required.

---

## Pre-flight checklist

- [ ] GitHub repo is public: https://github.com/sagarbpatel31/tracemind
- [ ] Vercel CLI installed (`vercel --version`)
- [ ] Anthropic API key ready (optional — enables LLM summaries)

---

## Step 1 — Supabase (database)

**Time: ~5 min**

1. Go to https://supabase.com → **Start your project** → sign up
2. **New project**
   - Name: `tracemind`
   - Database password: generate a strong one, save it
   - Region: **US West (North California)** — closest to Render Oregon
3. Wait ~2 min for provisioning
4. Go to **Settings → Database → Connection string → URI** tab
5. Copy the URI — it looks like:
   ```
   postgresql://postgres:[YOUR-PASSWORD]@db.[PROJECT-REF].supabase.co:5432/postgres
   ```
6. Save it — you'll paste this into Render as `DATABASE_URL`

> The `normalize_postgres_url()` validator in `config.py` rewrites this format automatically.
> Paste it exactly as Supabase gives it.

---

## Step 2 — Render (API backend)

**Time: ~8 min including build**

### 2a. Create account and connect repo

1. Go to https://render.com → **Get Started** → sign up with GitHub
2. **New → Blueprint**
3. Connect GitHub → select `sagarbpatel31/tracemind`
4. Render will detect `apps/api/render.yaml` and propose one service: `tracemind-api`
5. Click **Apply**

### 2b. Set environment variables

In the Blueprint env vars screen (before clicking Apply), set:

| Key | Value | Notes |
|-----|-------|-------|
| `DATABASE_URL` | `postgresql://postgres:[PASSWORD]@db.[REF].supabase.co:5432/postgres` | From Supabase Step 1 |
| `ANTHROPIC_API_KEY` | `sk-ant-...` | Optional — get from console.anthropic.com |

These are the only two marked `sync: false`. Everything else is pre-configured in `render.yaml`:

| Key | Value (pre-set in render.yaml) |
|-----|-------------------------------|
| `CORS_ORIGINS` | `https://tracemind.vercel.app,http://localhost:3000` |
| `JWT_SECRET_KEY` | Auto-generated by Render |
| `STORAGE_PATH` | `/tmp/storage` |
| `PORT` | `8000` |

### 2c. Wait for build and get URL

1. Build takes ~3–5 min (Docker build + pip install)
2. Once deployed, copy your service URL: `https://tracemind-api.onrender.com`
   (may differ — use whatever Render assigns)
3. Verify it's live:
   ```bash
   curl https://tracemind-api.onrender.com/api/v1/health
   # → {"status":"ok","service":"tracemind-api","version":"0.1.0"}
   ```
   > First request may take ~30s if service just started.

## Step 3 — Vercel (frontend)

**Time: ~3 min**

The frontend is already deployed at https://tracemind.vercel.app.
You only need to set one env var and redeploy.

```bash
cd /Users/sagarpatel/Documents/Tracemind/apps/web

# Add the production API URL (replace with your actual Render URL)
vercel env add NEXT_PUBLIC_API_URL production
# When prompted, enter: https://tracemind-api.onrender.com

# Redeploy to production with the new env var
vercel --prod
```

Alternatively via Vercel dashboard:
1. https://vercel.com → your `tracemind` project → **Settings → Environment Variables**
2. Add: `NEXT_PUBLIC_API_URL` = `https://tracemind-api.onrender.com` (scope: Production)
3. **Deployments → Redeploy** latest deployment

## Step 5 — Smoke test

Run these in order. All should pass.

```bash
export API=https://tracemind-api.onrender.com

# 1. Health
curl $API/api/v1/health
# → {"status":"ok","service":"tracemind-api","version":"0.1.0"}

# 2. Login
TOKEN=$(curl -s -X POST $API/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"demo@tracemind.ai","password":"demo123"}' \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")
echo "Token: ${TOKEN:0:20}..."

# 3. List incidents
curl -s $API/api/v1/incidents/ \
  -H "Authorization: Bearer $TOKEN" | python3 -m json.tool | grep '"title"'

# 4. Trigger analysis on first incident (get ID from step 3)
curl -s -X POST $API/api/v1/incidents/{INCIDENT_ID}/analyze \
  -H "Authorization: Bearer $TOKEN" | python3 -m json.tool | grep '"summary"'

# 5. Open frontend
open https://tracemind.vercel.app
# Login with demo@tracemind.ai / demo123
# Verify: dashboard loads, incidents show, analysis runs, replay bundle downloads
```

## Known limitations on free tier

| Limitation | Impact | Notes |
|-----------|--------|-------|
| Render cold start ~60s | First request after 15 min idle is slow | Add loading state on frontend |
| Supabase pauses after 1 week inactivity | ~30s resume lag | Resumes on first request automatically |
| `/tmp/storage` on Render is ephemeral | Replay bundles lost on restart/redeploy | Acceptable for demo; use S3 in production |
| Render free = 512MB RAM | Fine for this app | Watch if you add heavy ML inference |

## If something fails

**Render build fails:**
```bash
# Check build logs in Render dashboard → Events tab
# Common causes:
# - pip install failed (check pyproject.toml)
# - Dockerfile syntax error
# - Wrong rootDir in render.yaml (should be apps/api)
```

**API starts but DB errors:**
```bash
# Test DB connection
curl https://tracemind-api.onrender.com/api/v1/health
# If 500 → Render logs → likely DATABASE_URL wrong or Supabase not ready

# Verify URL format — Render env var should look like:
# postgresql://postgres:PASSWORD@db.XXXXX.supabase.co:5432/postgres
# (config.py rewrites it to postgresql+asyncpg:// automatically)
```

**Frontend shows no data after login:**
```bash
# Most likely: NEXT_PUBLIC_API_URL not set or wrong
# Verify in Vercel: Settings → Environment Variables
# Then: Deployments → Redeploy (must redeploy after adding env var)
```

**CORS errors in browser console:**
```bash
# Update CORS_ORIGINS in Render env vars to include your exact Vercel URL
# No trailing slash: https://tracemind.vercel.app NOT https://tracemind.vercel.app/
# Redeploy Render service after changing
```

**Seed returns 500 (already seeded):**
```bash
# DB already has demo data — this is fine, use the existing credentials
# If you need a fresh seed, run in Supabase SQL editor:
# TRUNCATE users, workspaces, projects, devices, deployments, incidents,
#   event_logs, metric_points, incident_artifacts, annotations CASCADE;
# Then re-run the seed curl
```
